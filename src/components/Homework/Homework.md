Задача 1
Что такое Formik и почему он используется в приложениях React?

Formik - это популярная библилотека для управления формами в Reat. Она
 используется для упрощения 3-х самых сложных задач при работе с формами:
 1.Управление состоянием значений формы(value).
 2.Валидация данных и отображение сообщений об ошибках.
 3.Обработка отправки формы (submission)

Задача 2
Какие задачи решает Formik по сравнению с работой с формами, используя
 только? useState

- ​С useState: Вам нужно создавать отдельный стейт для каждого инпута или сложный объект, вручную писать функции handleChange для каждого случая.
- ​С Formik: Все управление (ошибки, посещенные поля touched, значения values) инкапсулировано в одном месте.

Задача 3
Какова цель хука? useFormik()

Хук useFormik()  позволяет использовать функции и состояние Formik внутри
функционального компонента без использования обертки <Formik/>. Он возвращает
 объект со всеми необходимыми методами(handleChange, handleSubmit) и состоянием
 (value, errors), которые вы затем вручную привязываете к своим HTML-элементам.

🧩 Блок 3. Логика интеграции Formik
Задача 7
Почему атрибут элемента <input> должен совпадать с ключом в initialValues?

Formik использует атрибут name как идентификатор. Когда происходит событие onChange,
 Formik ищет значениет в объекте состояния по ключу, равному name инпута, чтобы обновить
именно это поле. Если они не совпадут, Formik не поймет, какое значение в values нужно
изменить.
name — это ключ. Именно через этот атрибут Formik связывает DOM-элемент со своим 
внутренним состоянием.

Задача 8
Что происходит, если у <input> отсутствует привязка value={formik.values.fieldName}?
(Свободный ответ)

 Инпут перестает быть контролируемым компонентом. Это значит, что если состояние
 Formik изменится программно ( например, через resetForm или setFeildValue), текст
 внутри инпута не обновится и будет отображать некорректные данные.
 Если вы программно очистите форму (например, через resetForm), текст в поле ввода 
 останется прежним, хотя в данных формы его уже нет. Это ведет к рассинхрону данных
 и интерфейса.

⭐ Блок 5. Вопрос по вызову
Задача 10
Почему важно использовать оба варианта (handleChange и handleSubmit) при работе с Formik?

- handleChange необходим для синхронизации: он обновляет состояние Formik в 
реальном времени, когда пользователь вводит текст. Без него вы не сможете
получить данные из полей.
- handleSubmit необходим для завершения процесса: он предотвращает стандартное 
поведение браузера  (перезагрузку страницы), запускает валидацию и вызывает вашу 
логику отправки данных на сервер только в том случае, если форма заполнена верно.
